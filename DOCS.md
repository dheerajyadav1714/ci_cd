### **Pipeline Endpoints**

Pipelines represent the core CI/CD workflows.

#### 1. List All Pipelines

Retrieves a list of all defined CI/CD pipelines.

*   **Endpoint:** `GET /pipelines`
*   **Description:** Returns an array of pipeline objects.
*   **Query Parameters:**
    *   `limit` (optional, integer): Maximum number of pipelines to return. Default: 20.
    *   `offset` (optional, integer): Number of pipelines to skip before starting to collect the result set. Default: 0.
    *   `status` (optional, string): Filter pipelines by their current status (e.g., `active`, `paused`).
*   **Response:** `200 OK`
    ```json
    {
      "pipelines": [
        {
          "id": "pipe-123",
          "name": "Frontend Application CI",
          "description": "Build and test for the main web application.",
          "status": "active",

        },

          "id": "pipe-456",
    ```

#### 2. Create a New Pipeline

Defines and registers a new CI/CD pipeline.

*   **Endpoint:** `POST /pipelines`
*   **Description:** Creates a new pipeline definition.
*   **Request Body:** `application/json`
    *   `name` (required, string): Unique name for the pipeline.
    *   `description` (optional, string): A brief description of the pipeline's purpose.
    *   `repositoryUrl` (required, string): URL of the source code repository (e.g., Git URL).
    *   `pipelineConfig` (required, object): A JSON object defining the pipeline stages and steps. This would be specific to the internal pipeline definition language (e.g., YAML-like structure).
        ```json
        {
          "name": "New API Service Pipeline",
          "description": "Build, test, and deploy new API service.",
          "repositoryUrl": "https://github.com/yourorg/api-service.git",
          "pipelineConfig": {
            "stages": [
              {
                "name": "Build",
                "steps": [
                  {"command": "npm install"},
                  {"command": "npm run build"}
                ]
              },
              {
                "name": "Test",
                "steps": [
                  {"command": "npm run test"}
                ]
              },
              {
                "name": "Deploy Staging",
                "steps": [
                  {"command": "deploy.sh staging"}
                ],
                "manualApproval": true
              }
            ]
          }
        }
        ```
*   **Response:** `201 Created`
    ```json
    {
      "id": "pipe-789",
      "name": "New API Service Pipeline",
      "status": "active",
      "createdAt": "2023-10-26T14:30:00Z"
    }
    ```
    *   `409 Conflict`: If a pipeline with the same name already exists.

    ```json
      "status": "active",
    ```

#### 4. Trigger a Pipeline Run

Manually triggers a new execution of a specific pipeline.

*   **Endpoint:** `POST /pipelines/{pipelineId}/trigger`
*   **Description:** Initiates a new run for the specified pipeline.
*   **Path Parameters:**
    *   `pipelineId` (required, string): The unique identifier of the pipeline.
*   **Request Body:** `application/json` (optional)
    *   `branch` (optional, string): The Git branch to build/deploy from. Default: `main`.
    *   `variables` (optional, object): Key-value pairs for environment variables or parameters specific to this run.
    ```json
    {
      "branch": "feature/new-ui",
      "variables": {
        "DEPLOY_ENV": "staging",
        "SKIP_TESTS": "true"
      }
    }
    ```
*   **Response:** `202 Accepted`
    ```json
    {
      "message": "Pipeline run triggered successfully.",
      "runId": "run-xyz",
      "pipelineId": "pipe-123"
    }
    ```
    *   `404 Not Found`: If the `pipelineId` does not exist.
    *   `409 Conflict`: If the pipeline is currently paused or in a state that prevents triggering.
